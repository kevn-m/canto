# Chibi mascot sprite pipeline

All sprites in this folder are generated by `sprites.js` — a shape-recipe system, not hand-placed pixels. To add subjects, append recipes to `sprites.js`; never draw pixel grids by hand or use image tools. Export locally with `node render.js` (writes `<name>.png` for every recipe and audits the palette).

## Spec (hard rules — repeat-check every batch)
- 64×64 PNG, transparent background, one centred subject filling ~70–80%
- PICO-8 palette only (defined as `P` in sprites.js); audit after export
- 1px navy `#1D2B53` outline (automatic pass), never black
- Chibi: head ≈ half sprite height, no neck, stubby limbs
- Exaggerate exactly 2 signature features; simplify everything else
- Mouth turns UP, always visible. Eyes large navy ovals + cream sparkle, set low/wide. Optional pink blush.
- No text, nothing scary, must pass silhouette test (render with `drawTo(ctx, name, true)`)
- Light from top-left; one shadow tone + max one highlight tone per surface; no dithering/gradients/AA

## The one exception: FX sprites (`fx-*`)

Battle effects — `fx-slash`, `fx-impact` — are the only sprites with no face and no
chibi armature. They are shapes, not creatures; eyes on a slash arc would be absurd.
Palette, outline and no-AA still apply. Everything else in the spec above does not.

## The kit (in sprites.js)
- `ball(g,cx,cy,rx,ry,base,shad,hi)` — self-shading ellipse (shadow rim bottom-right, light top-left). The workhorse.
- `ellipse/disc/rect/rrect/tri` — flat fills
- `stroke(g,pts,r0,r1,c)` — tapered polyline of discs: trunks, tails, steam. For a limb that needs its own outline against same-colour body, stroke navy at r+1 first, then the colour (see elephant trunk).
- `clipTo(g,[colors],fn)` — paint only over listed colours: stripes, patches
- `eyes/smileArc/openMouth/blush` — face kit; `smileArc(g,cx,y,w,depth)` ends curve UP
- `outline(g)` — auto navy dilation, runs last; erasing with `null` (bite notches, handle holes) gets outlined correctly
- `CX = 31.5` — centre; use for all symmetric placement

## Standard armature (deviate only for signature features)
head ball (CX, 25, 15, 13) · body ball (CX, 48, ~10, 8) · feet balls at x 25/38, y 56 · eyes y≈25 spread 6–8 · muzzle cy≈31 · smile y≈32 · blush at spread+5, y≈30. Draw order: back features (ears/tail/fins) → body → limbs → head → face.

## Colour ramps (base → shadow, optional highlight)
sky→lav · lgy→lav (hi crm) · org→brn (hi yel) · yel→org (hi crm) · brn→plum · lim→grn · red→plum (hi pnk) · crm→lgy. Keep highlights ONLY on low-contrast pairs — high-contrast highlights read as blotches (learned: removed from dolphin/bear/croc).

## Workflow for a new batch
1. Append recipes to `S` in sprites.js (name + draw fn, ~15 lines each).
2. `node render.js` — regenerates every PNG in this folder and fails loud on any out-of-palette colour.
3. Iterate coordinates 1–3 rounds. Common fixes: detached appendages (move within 1px of body), same-colour features vanishing (navy underlay stroke), highlight blotches (remove hi).
4. Silhouette-check new sprites, then get the batch approved before bundling into the app.
